# 🍴 식당 리뷰 작성 프로그램

## 프로젝트 개요

터미널 기반의 식당 방문 후기 수집 시스템입니다.
고객이 식사 후 간단한 설문 방식으로 리뷰를 남기고, 누적된 통계를 조회할 수 있습니다.
MySQL 데이터베이스와 연동하여 데이터를 영구적으로 저장·관리합니다.

---

## 기술 스택

## DB 테이블 구조

| 컬럼 | 설명 |
|------|------|

| count | 선택 횟수 |

## 주요 기능

### 1. 리뷰 작성
고객이 3가지 질문에 순서대로 답변하면, 각 항목의 `count` 값이 DB에서 +1 증가합니다.

- **Q1. 어떻게 이용하셨나요?** (이용 방법 선택)
- **Q2. 누구와 함께 방문하셨나요?** (동반자 유형 선택)
- **Q3. 좋았던 점 한 가지만 선택해 주세요!** (장점 선택)

### 2. 리뷰 보기
누적된 응답 데이터를 건수 기준 내림차순으로 순위를 매겨 테이블 형식으로 출력합니다.

```
--- 리뷰 보기 ---

┌────────────────────────────────────────────────┐
| 순위  |       이용 방법       |  건수 |
├────────────────────────────────────────────────┤
|  1    |      예약 후 이용     |   9   |
|  2    |    예약 없이 이용     |   6   |
|  3    |    포장/배달 이용     |   5   |
...
└────────────────────────────────────────────────┘
```

## 실행 방법

### 1. 패키지 설치
```bash
pip install pymysql
```

### 2. DB 접속 정보 설정
`review.py` 내 연결 정보를 환경에 맞게 수정합니다.
```python
conn = pymysql.connect(
    host="192.168.32.22",
    port=3306,
    user="testuser",
    password="test1234",
    database="mydb",
    charset="utf8mb4"
)
```

### 3. 프로그램 실행
```bash
python review.py
```

## 특이사항 및 구현 포인트

- 한글 문자열을 터미널에서 정렬하기 위해 `center_kor()` 함수를 직접 구현하여 사용
  - 한글은 영문 대비 2배 너비를 차지하므로, 유니코드 범위 기준으로 패딩을 별도 계산
- 리뷰 선택 시 DB에 새 행을 삽입하는 방식이 아닌, 기존 항목의 `count`를 `UPDATE`하는 방식으로 집계
- `try / finally` 구조로 DB 연결이 항상 정상적으로 종료되도록 처리
